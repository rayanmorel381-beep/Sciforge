## 2️⃣ Framebuffer — `src/render/`

| Component | Description |
|---|---|
| `Framebuffer` | `{ width: u32, height: u32, pixels: Vec<u32> }` RGBA packed |
| Depth buffer | `Vec<f32>` for z-buffering |
| `clear(color)` | Clear the framebuffer |
| `set_pixel(x, y, z, color)` | Write pixel with depth test |
| `get_pixel(x, y)` | Read pixel |
| `blend_pixel(x, y, color, alpha)` | Alpha blending |
| BMP/PPM encoding | Native encoding (no dependency) for raw export |
| base64 data URI | Direct HTML injection via existing base64_encode |

## 3️⃣ Camera & Projection

| Component | Description |
|---|---|
| `Camera` | `{ position: Vec3, target: Vec3, up: Vec3, fov: f64, near: f64, far: f64, aspect: f64 }` |
| `view_matrix()` | → `[[f64;4];4]` (lookAt) |
| `projection_matrix()` | → `[[f64;4];4]` (perspective) |
| `viewport_transform(w, h)` | → `[[f64;4];4]` |
| `project_point(world)` | → `Option<(i32, i32, f64)>` screen coordinates + depth |
| Quaternion | Reuse of from_euler, rotate_vector for camera orientation |

## 4️⃣ 3D Rasterization Pipeline

| Component | Description |
|---|---|
| `Vertex` | `{ position: Vec3, color: u32, normal: Vec3 }` |
| `Triangle` | `(Vertex, Vertex, Vertex)` |
| Transformation | model × view × projection × viewport (4×4 matrix multiplication) |
| Clipping | Near/far planes |
| Rasterization | Scanline with barycentric interpolation |
| Z-buffer | Per-pixel depth test |
| Phong lighting | Ambient + diffuse (dot(N, L)) + specular (dot(R, V)^shininess) |
| Shading | Flat shading + Gouraud shading (per-vertex color interpolation) |

## 5️⃣ Geometric Primitives

| Function | Signature | Description |
|---|---|---|
| Sphere | `sphere(center, radius, subdivisions) → Vec<Triangle>` | Icosphere or UV-sphere |
| Cube | `cube(center, size) → Vec<Triangle>` | Axis-aligned cube |
| Cylinder | `cylinder(base, top, radius, segments) → Vec<Triangle>` | Cylinder |
| Plane | `plane(center, normal, size) → Vec<Triangle>` | Plane |
| Axes gizmo | `axes_gizmo(origin, scale) → Vec<Triangle>` | RGB X/Y/Z axes |
| Arrow | `arrow(from, to, radius) → Vec<Triangle>` | Velocity vectors |
| Orbit trail | `orbit_trail(points) → Vec<Triangle>` | Tube or thickened line |

## 6️⃣ 5D Extension (Time + Extra Dimension)

| Component | Description |
|---|---|
| `extra_dims` | Optional field `Vec<[f64;2]>` on SimState |
| 4th dimension (time) | Slice time into frames, each frame = 3D cross-section of spacetime |
| 5th dimension | Stereographic projection or color projection (dim5 → HSL hue) |
| N-D rotations | Reuse of Tensor for 4D/5D rotation matrices |
| Curved trajectories | Reuse of GeodesicSolver visualized in 3D |
| 5D→3D projection | Perspective projection + color/size mapping for the 2 extra dimensions |

## 7️⃣ RenderSink Implementation

| Component | Description |
|---|---|
| `FramebufferSink` | `{ fb: Framebuffer, camera: Camera, scene: Vec<Triangle> }` |
| `impl RenderSink` | On each frame(SimState), transform positions → 3D primitives → rasterize → store |
| Single frame | `render_to_bmp(path)` |
| Sequence | `render_frames() → Vec<Framebuffer>` for animation |
| Binary streaming | Write each encoded frame to a buffer/file |

## 8️⃣ HTML Integration

| Option | Description |
|---|---|
| Image inline | `<img src="data:image/bmp;base64,...">` per frame via existing base64_encode |
| JS blob | Raw pixels → ImageData + putImageData on canvas |
| WebGL vertex buffer | Export vertices as binary from Rust, load on JS side |
| Upgrade simulations | Orbit, projectile, etc. from Canvas 2D → real 3D rendering |

## 9️⃣ Concrete Scenes

| Scene | Description |
|---|---|
| 3D orbit | Spherical planet + orbiting body + trail + axes |
| 3D projectile | Flat terrain + 3D parabolic trajectory + ground shadow |
| Nucleosynthesis | Particles/nuclei as colored spheres + reaction trajectories |
| 3D vector field | Arrows from VectorFieldSim |
| Curved spacetime | Deformed grid via GeodesicSolver + particle following a geodesic |
| 5D visualization | Relativistic orbit (Lorentz boost) dim4=time dim5=energy as color |

## 🔟 Build & Performance

| Constraint | Description |
|---|---|
| Pure Rust | Zero external dependency |
| CPU rasterization | Software rendering, no GPU required |
| SIMD (optional) | `#[cfg(target_arch = "x86_64")]` for scanline and matrix ops |
| Benchmarking | Via existing BenchmarkMetrics (ns/frame, pixels/s) |
| Target resolution | 800×600 minimum, configurable |
